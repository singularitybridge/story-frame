Story Edit API Implementation

Documentation of the dual-agent story editing system migration

Goal: Replace /api/chat with /api/story/edit for single-shot story editing using dual-agent system (Script Editing Agent + Review Agent)

User Feedback & Requirements

1. Endpoint Naming

User suggested renaming /api/chat to /api/story/edit to better reflect the single-shot story editing behavior:

  • Old: /api/chat - Implied conversational, multi-turn experience
  • New: /api/story/edit - Clear single-shot editing endpoint
  • Rationale: "Chat" endpoint might evolve for chat-like experience later, while story editing needs focused, formatted responses

2. Why Story Generation Works

Answer: /api/story/generate uses @google/genai package directly, not @ai-sdk/google

  • @google/genai - Official Google SDK, stable and well-tested
  • @ai-sdk/google - Vercel AI SDK wrapper with tool calling issues

3. Development Strategy Guidelines

  1. Always prefer to test via API/shell/curl calls
  2. If encountering issues, isolate and test with small test scripts
  3. Once working, apply changes to the codebase
  4. Test again via API/shell/curl
  5. Test via UI using Chrome DevTools MCP
  6. Store ideas and useful info in README.md and CLAUDE.md files
  7. Update CLAUDE.md to follow guidelines
  8. LLM prompts must be saved in .agents/agent-name/prompt.md and loaded from there in code

Technical Issues Discovered

Issue #1: API Key Error

Error: Google Generative AI API key is missing

Root Cause: @ai-sdk/google requires GOOGLE_GENERATIVE_AI_API_KEY env var, not NEXT_PUBLIC_GEMINI_API_KEY

Fix: Added GOOGLE_GENERATIVE_AI_API_KEY to .env.local

Issue #2: Tool Schema Error

Unable to submit request because `edit_script` functionDeclaration parameters schema should be of type OBJECT

Root Cause: Incompatibility between Zod schemas and @ai-sdk/google provider for Gemini function calling

Attempts:

  • ✗ Tried using tool() helper - same error
  • ✗ Tried different Zod schema formats - same error
  • ✗ Checked Zod version (4.1.12 latest) - correct version

Conclusion: AI SDK tool calling with Gemini has unresolved compatibility issues

Solution: Use @google/genai Directly

Created /api/story/edit endpoint using @google/genai directly, matching the proven approach from /api/story/generate

Implementation

New API Endpoint: /api/story/edit

File: app/api/story/edit/route.ts

Request Format:

{
  "storyDraft": {
    "projectMetadata": { /* project metadata */ },
    "scenes": [ /* array of scenes */ ]
  },
  "editRequest": "rename the hero to Maria"
}

Response Format:

{
  "updatedStory": { /* modified StoryDraft */ },
  "response": "Okay, I've renamed the hero to Maria...",
  "changesSummary": {
    "scenesAdded": 0,
    "scenesRemoved": 0,
    "scenesModified": 1,
    "titleChanged": false
  }
}

UI Updates: CreateStoryModal.tsx

Changes:

  • Removed useChat() from @ai-sdk/react
  • Replaced with simple async editStory() function using fetch()
  • Simplified message state management
  • Updated form submission handlers

Before (useChat hook):

const {messages, sendMessage, status} = useChat({
  api: '/api/chat',
  onFinish: (message) => {
    // Extract updated story from tool invocations
    if (message.toolInvocations?.[0]?.result?.updatedStory) {
      setStoryDraft(message.toolInvocations[0].result.updatedStory);
    }
  },
});

After (simple fetch):

const editStory = async (editRequest: string) => {
  const response = await fetch('/api/story/edit', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({storyDraft, editRequest}),
  });

  const result = await response.json();
  setStoryDraft(result.updatedStory);
  setMessages(prev => [...prev, {
    role: 'assistant',
    content: result.response
  }]);
};
